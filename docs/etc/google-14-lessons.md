## 핵심 요약

Osmani는 이전에 21가지 교훈을 공유한 바 있는데, 가장 반향이 컸던 조언은 기술이 아닌 **사람과 협업**에 관한 것이었다고 한다. 이 후속편은 팀 역학에 집중한다: **의사결정이 실제로 어떻게 이뤄지는지, 어디서 협업이 무너지는지, 성과를 내는 팀과 제자리에서 맴도는 팀의 차이**가 핵심이다.

### 14가지 교훈 한 줄 요약

| # | 교훈 | 핵심 |
|---|------|------|
| 1 | 최고의 엔지니어는 **올바른 문제**를 고른다 | 빠른 게 아니라 선별적인 것 |
| 2 | 어떤 **결정**을 요청하는지 말할 수 없으면 회의할 준비가 안 된 것 | 회의 시작 2분 안에 ask 명시 |
| 3 | "우리 해야 해"는 계획이 아니다. **"화요일에 내가 할 것"**이 계획이다 | 이름+날짜 = 모멘텀 |
| 4 | 느린 코드는 증상, **느린 의사결정**은 항상 문제 | 권한이 명확하면 결정은 빠르다 |
| 5 | 안정성은 **제품 기능**이다 | Error budget으로 트레이드오프 명시 |
| 6 | 커뮤니케이션으로 **팀 간 나쁜 인터페이스**를 극복할 수 없다 | 회의 추가가 아닌 경계 설계 |
| 7 | 최고의 에스컬레이션에는 **제안**이 포함된다 | "도와주세요" → "A와 B 중 B 추천" |
| 8 | 히어로 문화를 피하라. **영웅이 필요 없는 시스템**을 만들어라 | 평범한 화요일을 위해 설계 |
| 9 | Observability를 **기능의 일부**로 만들어라 | 텔레메트리 없는 기능은 숨겨진 부채 |
| 10 | 작은 PR은 **친절**이다 | 작은 변경 = 빠른 리뷰 + 독립 롤백 |
| 11 | 팀을 추가하면 노드가 아닌 **엣지**가 늘어난다 | 인원 증가 ≠ 산출 증가 |
| 12 | 마이그레이션은 **절대 단순한 마이그레이션이 아니다** | 기술보다 인적 작업이 크다 |
| 13 | AI가 초안을 싸게 만들었다. **안목(Taste)**이 비싸졌다 | 무엇을 빼고 남길지가 차별화 |
| 14 | 신뢰는 팀의 **지연시간 최적화**다 | 약속 이행이 의사결정 속도를 높인다 |

## 원편: 21 Lessons from 14 Years at Google

위 14가지 교훈의 **원편**으로, 기술적 판단과 커리어 전반에 걸친 21가지 교훈을 다룬다.

### 21가지 교훈 한 줄 요약

| # | 교훈 | 핵심 |
|---|------|------|
| 1 | 최고의 엔지니어는 **사용자 문제 해결**에 집착한다 | 기술이 아니라 문제 이해가 먼저 |
| 2 | 옳은 것은 싸다. **함께 옳은 답에 도달**하는 게 진짜 일이다 | 논쟁에서 이기기보다 합의 도출 |
| 3 | 실행 편향. 출시하라. **빈 페이지는 편집할 수 없다** | 완벽 대기보다 피드백 기반 개선 |
| 4 | 명확함이 시니어리티. **영리함은 오버헤드** | 이해하기 쉬운 코드가 진짜 가치 |
| 5 | 새로움은 **장애·채용·인지 부하로 갚는 빚**이다 | 새 기술 선택 = 운영 비용 증가 |
| 6 | 코드는 당신을 대변하지 않는다. **사람이 한다** | 부재 시에도 인정받는 관계 구축 |
| 7 | 최고의 코드는 **쓸 필요가 없었던 코드** | 제거가 추가보다 큰 개선 |
| 8 | 규모에서는 **버그에도 사용자가 있다** | 모든 동작이 의존 대상 → 호환성 중요 |
| 9 | 대부분의 "느린" 팀은 사실 **방향이 안 맞는 팀** | 기술이 아니라 우선순위 불일치 |
| 10 | 통제 가능한 것에 집중. **불가능한 것은 무시** | 영향력 범위 안에 에너지 집중 |
| 11 | 추상화는 복잡성을 제거하지 않는다. **온콜 날로 옮길 뿐** | 기저 원리 이해 필수 |
| 12 | 글쓰기는 명확함을 강제한다. **가르치는 것이 가장 빠른 학습** | 설명 시도 → 이해 부족 발견 |
| 13 | 다른 일을 가능하게 하는 일은 **값을 매길 수 없지만 보이지 않는다** | 글루 워크의 의도적 관리 필요 |
| 14 | 모든 논쟁에서 이기면 **조용한 저항이 쌓인다** | 표면적 동의 ≠ 실행 단계 협조 |
| 15 | 측정이 목표가 되면 **측정이 아니게 된다** | Goodhart's Law - 지표 조작 위험 |
| 16 | 모르는 것을 인정하면 **아는 척보다 안전한 문화**가 만들어진다 | 리더의 불확실성 인정 → 학습 문화 |
| 17 | 네트워크는 **어떤 직장보다 오래 간다** | 관계 관리 = 커리어 최대 자산 |
| 18 | 대부분의 성능 향상은 영리함이 아닌 **작업 제거**에서 온다 | 불필요한 것 제거가 최적화 |
| 19 | 프로세스는 문서 흔적이 아닌 **불확실성 감소**를 위해 존재 | 좋은 프로세스 ≠ 관료주의 |
| 20 | 결국 시간이 돈보다 가치 있어진다. **그에 맞게 행동하라** | 경력 후반 = 시간이 최고 자원 |
| 21 | 지름길은 없다. 하지만 **복리 효과는 있다** | 장기 의도적 연습 → 선택지 확장 |
