## 공통 관측 프레임워크

RSI와 달리 MACD는 "레벨(과열/과매도)"보다 **"모멘텀의 방향과 가속도"**를 읽는 도구입니다. 이 차이를 이해하는 것이 전략 설계의 출발점입니다.

### MACD 구성 요소별 역할

| 구성 요소 | 읽는 것 | 실전 활용 |
|-----------|---------|-----------|
| **MACD선** (단기 EMA - 장기 EMA) | 추세 모멘텀의 방향 | 0선 위/아래 → 상승/하락 압력 우위 |
| **시그널선** (MACD의 EMA) | 모멘텀의 평활화 | MACD가 시그널 위/아래 → 단기 강약 |
| **히스토그램** (MACD - 시그널) | 모멘텀의 **가속/감속** | 가장 먼저 변하는 선행 지표 |

### MA 배열 체크포인트

| 항목 | 확인 내용 |
|------|-----------|
| **정렬** | 단기 > 중기 > 장기 (상승) / 반대 (하락) |
| **기울기** | 장기 MA 우상향 여부 → 진짜 추세의 핵심 |
| **이격** | 가격과 단기 MA 간 거리 → 과열/되돌림 위험 |

### 마라톤 선수 비유

- **가격** = 선수의 현재 위치 (42km 중 지금 몇 km 지점)
- **MA** = 코스의 방향 (지금 오르막 구간인지, 내리막 구간인지 — 레짐)
- **MACD** = 속도계 + 가속도 (지금 페이스가 빨라지고 있는지, 떨어지고 있는지)
- **RSI** = 심박수 / 체력 잔량 (과호흡 직전인지, 아직 여유가 있는지)

RSI는 가격·MA·MACD와 같은 위치-속도-가속도 계층이 아닌, 완전히 다른 축의 정보입니다. 선수가 오르막(하락 레짐)에서 전력 질주(강한 하락 모멘텀)하고 있어도 심박수가 아직 여유 있으면 더 버틸 수 있고, 반대로 내리막(상승 레짐)인데 이미 심박수가 레드존이면 곧 무너질 수 있습니다. RSI가 "지금 이 움직임이 생리적으로 지속 가능한가"를 보는 지표인 이유가 이 비유에서 자연스럽게 나옵니다.

### MACD 핵심 신호 4가지

| 순번 | 신호 | 설명 |
|------|------|------|
| 1 | **골든/데드크로스** | MACD선이 시그널선을 상향/하향 돌파 |
| 2 | **0선 크로스** | MACD선이 0선을 상향/하향 돌파 (레짐 전환의 확인) |
| 3 | **히스토그램 방향 전환** | 모멘텀 가속/감속의 가장 빠른 신호 (선행) |
| 4 | **다이버전스** | 가격과 MACD의 방향 불일치 (전환 경고) |

> MACD 사용 시 가장 흔한 실수: **골든크로스만 보고 진입**하는 것. 0선 위치와 히스토그램 방향을 같이 보지 않으면 추세 초입인지 막바지인지 구분이 불가능합니다.

## 케이스 1. 정상 상승 추세

> **"MACD가 0선 위에서 살아있는 동안은 추세 추종이 정답"**

| 구분 | 조건 |
|------|------|
| MA | 단기 > 중기 > 장기 정렬 + 장기 MA 우상향 |
| MACD선 | **0선 위** 유지 |
| 시그널선 | MACD선이 시그널선 위 또는 근접 |
| 히스토그램 | 양수(+) 유지, 눌림 때 줄어들다가 재확대 |

### 진입 시점

- **1차 진입**: 눌림 조정에서 히스토그램이 축소 → 재확대로 전환되는 시점 + 중기 MA 지지 확인
- **2차 추가**: MACD 골든크로스 + 히스토그램이 다시 양수 확대 국면 ("모멘텀 재점화")

### TP/SL

| 구분 | 조건 |
|------|------|
| **SL** | 중기 MA 종가 이탈 + MACD선이 시그널선 하향 이탈 **동시** 충족 |
| **TP 1** | 직전 스윙 고점 부근 분할 청산 |
| **TP 2** | 히스토그램이 고점을 낮추며 수축 시작 (모멘텀 감속 신호) → 일부 정리 |
| **TP 3** | 추세 유지형이면 **중기 MA 기반 추적 손절**로 수익 극대화 |

## 케이스 2. 상승 추세의 과열/피로

> **"MACD가 0선 위 고점권에서 히스토그램이 수축하기 시작하면 경고"**

| 구분 | 조건 |
|------|------|
| MA | 정렬은 상승, 가격이 단기 MA에서 과도 이격 |
| MACD선 | 0선 위이지만 고점권에서 횡보 또는 소폭 하락 |
| 히스토그램 | **양수이지만 점점 수축** (막대가 작아지는 중) |
| 다이버전스 | 베어리시 다이버전스 출현 가능성 (가격 HH, MACD LH) |

### 실전 운영

| 포지션 | 대응 |
|--------|------|
| **신규 진입** | 보수적 대기. 히스토그램이 재확대될 때까지 추격 금지 |
| **기존 보유** | 히스토그램 수축 구간에서 분할 익절 시작 |
| **SL 조정** | 단기 MA 이탈이 아닌 **MACD 데드크로스 + 중기 MA 이탈** 조합으로 상향 |

## 케이스 3. 상승 추세 → 분배/전환 초기 신호

> **"히스토그램이 먼저 말하고, MACD 0선 하향이 확인해준다"**

전환도 단계적으로 읽어야 합니다.

| 단계 | 신호 |
|------|------|
| 1단계 | **히스토그램 베어리시 다이버전스** (가격 HH, 히스토그램 LH) ← 가장 먼저 나타남 |
| 2단계 | MACD 데드크로스 (MACD선이 시그널선 하향 이탈) |
| 3단계 | 단기 MA 기울기 둔화 → 단기/중기 데드크로스 |
| 4단계 | **MACD선이 0선 하향 이탈** ← "전환 확인"의 가장 중요한 신호 |

### 진입 시점

- **숏/헤지**: MACD 데드크로스 + 0선 하향 이탈 + 단기 < 중기 MA 첫 확인 시
- **기존 롱 보유자**: 1~2단계 신호에서 포지션 축소 시작, 4단계에서 나머지 정리

### TP/SL

| 구분 | 조건 |
|------|------|
| **SL** | 마지막 구조적 고점 상향 돌파 + MACD가 0선 위로 재진입 시 |
| **TP** | 장기 MA(120/200) 부근 또는 이전 박스 하단까지 단계적 목표 |

---

## 케이스 4. 정상 하락 추세

> **"MACD가 0선 아래에서 데드크로스를 반복하는 동안은 역추세 금지"**

| 구분 | 조건 |
|------|------|
| MA | 단기 < 중기 < 장기 정렬 + 장기 MA 하향 |
| MACD선 | **0선 아래** 유지 |
| 히스토그램 | 음수(-) 유지, 반등 때 수축하다가 재확대 |

### 전략 분기

| 전략 | 조건 |
|------|------|
| **추세 추종 (숏/현금 확대)** | 반등 시 히스토그램이 0선 근처까지 수축 후 음수 재확대 시점 |
| **역추세 반등 매매** | MACD가 0선 아래에서 골든크로스만 나온 것은 **부족**. 케이스 5-B 조건 필요 |

### TP/SL

| 구분 | 조건 |
|------|------|
| **숏 TP** | 히스토그램 음수 확대가 멈추고 수축 전환 + 이전 저점 도달 |
| **숏 SL** | MACD선 0선 상향 회복 + 중기 MA 회복 동시 |

---

## 케이스 5. 하락 추세 속 반등 — 함정 vs 바닥 전환

> **"MACD가 0선을 넘느냐가 가장 결정적인 갈림길"**

### 5-A. 데드캣 바운스 (반등 함정)

| 항목 | 상태 |
|------|------|
| MA | 장기 하향, 배열 하락 정렬 유지 |
| MACD선 | 골든크로스 발생했지만 **0선 아래**에서 머무름 |
| 히스토그램 | 음수에서 양수로 전환됐지만 소폭, 다시 수축 시작 |
| 가격 | 중기 MA 아래에서 반등 종료 |

→ **"0선 아래 골든크로스"는 약한 신호**. 하락 추세 내 조정 가능성 높음.

### 5-B. 진짜 바닥 전환 후보

| 항목 | 상태 |
|------|------|
| MACD | 불리시 다이버전스 (가격 LL, MACD/히스토그램 HL) |
| 결정적 확인 | **MACD선이 0선 상향 돌파** |
| 히스토그램 | 양수로 전환 후 **확대** 지속 |
| MA | 단기 MA 평평→상향 전환, 가격이 중기 MA 회복 후 지지 |

**핵심 체크리스트:**
- 히스토그램 불리시 다이버전스 확인
- MACD 골든크로스 발생
- **MACD선 0선 상향 돌파** ← 가장 중요
- 가격이 중기 MA 위에서 지지

### TP/SL

| 구분 | 조건 |
|------|------|
| **SL** | 다이버전스 저점 이탈 + MACD선 0선 재하향 동시 |
| **TP 1차** | 중기/장기 MA 구간 (첫 저항) |
| **TP 이후** | 케이스 1 (정상 상승 추세) 운영으로 전환 |

## 케이스 6. 레인지/박스 장

> **"MACD는 레인지에서 잡음이 많다 — 반드시 MA 박스 경계와 같이 써야 한다"**

RSI와의 가장 큰 차이: RSI는 레인지 장에서 과매수/과매도 레벨이 명확해 단독으로도 잘 작동하지만, **MACD는 레인지에서 골든/데드크로스가 빈번하게 발생해 잡음이 훨씬 많습니다.**

| 구분 | 조건 |
|------|------|
| MA | 장기 MA 평평, 단기/중기 MA 엉킴 |
| MACD선 | 0선 근처에서 위아래 반복 |
| 히스토그램 | 양수/음수 작은 폭으로 반복 전환 |

### 실전 운영

MACD 단독 크로스 신호는 **무시** (잡음 처리). 대신 **가격이 박스 경계에 도달 + 히스토그램 방향 전환**을 조합으로 사용합니다.

| 조건 | 매매 |
|------|------|
| 박스 하단 + 히스토그램이 음수에서 양수로 전환 | 매수 |
| 박스 상단 + 히스토그램이 양수에서 음수로 전환 | 매도 |

### TP/SL

| 구분 | 조건 |
|------|------|
| **TP** | 박스 반대편 도달 시 |
| **SL** | 박스 종가 이탈 + MACD선이 0선을 한 방향으로 명확히 돌파 (레짐 변화 신호) |

## 케이스 7. 돌파 장 (레인지 → 추세 레짐 전환)

> **"MACD 0선 돌파가 레인지 이탈을 '진짜 돌파'로 확인해주는 열쇠"**

### 돌파 판별 조건

| 단계 | 상태 |
|------|------|
| **돌파 전** | MA 평평/엉킴 + MACD 0선 근처 횡보 |
| **돌파 발생** | 가격이 중기/장기 MA 위 안착 + MACD 골든크로스 |
| **결정적 확인** | **MACD선이 0선 상향 돌파 + 히스토그램 양수 확대** ← 핵심 |
| **리테스트** | 눌림에서 MACD선이 0선 위 유지 + 히스토그램 수축 후 재확대 |

### 진입 시점

| 스타일 | 조건 |
|--------|------|
| **보수적** | 리테스트에서 MACD가 0선 위 유지 + 히스토그램 재확대 확인 후 진입 |
| **공격적** | MACD 골든크로스 + 0선 돌파 동시 발생 시 진입 |

### TP/SL

| 구분 | 조건 |
|------|------|
| **SL** | 리테스트 실패 + MACD선 0선 재하향 |
| **TP 1차** | 직전 레인지 높이만큼 측정 목표 |
| **TP 이후** | 케이스 1 (정상 상승 추세) 운영으로 전환 |

## TP/SL 규칙화 요약

| 방식 | 적합 케이스 | 설명 |
|------|------------|------|
| **히스토그램 기반** (수축/확대 방향 전환) | 케이스 1, 2, 4 | **가장 빠른 선행 신호**, TP 타이밍에 유리 |
| **0선 기반** (MACD선 0선 크로스) | 케이스 3, 5, 7 | 레짐 전환 확인용, SL/진입 조건에 유리 |
| **크로스 기반** (골든/데드크로스) | 케이스 1, 3, 4 | 단독 사용 금지, MA 배열과 반드시 조합 |
| **다이버전스 기반** | 케이스 2, 3, 5 | 전환 경고 신호, 확인 신호와 반드시 조합 |

## 레짐 분류 빠른 판단 트리

```
장기 MA 방향은?
├─ 우상향 → 상승 레짐
│       MACD 0선 위 + 히스토그램 확대?  → 케이스 1 (추세 추종)
│       히스토그램 수축 + 다이버전스?   → 케이스 2/3 (과열/전환 주의)
│
├─ 하향 → 하락 레짐
│       MACD 0선 아래 골든크로스만?    → 케이스 5-A (함정, 진입 금지)
│       다이버전스 + 0선 상향 돌파?    → 케이스 5-B (전환 후보)
│
└─ 횡보/평평 → 레인지 레짐
        MACD 0선 근처 횡보?           → 케이스 6 (크로스 무시, 박스 경계 활용)
        MACD 0선 돌파 + 히스토그램 확대? → 케이스 7 (돌파 대기/확인)
```

## RSI vs MACD 비교 — 상황별 선택 기준

| 상황 | RSI | MACD |
|------|-----|------|
| **레인지 장** | 과매수/과매도 레벨 명확 (유리) | 잡음 많음 (불리) |
| **추세 초입 확인** | 50선 돌파로 판단 (보통) | 0선 돌파가 더 명확 (유리) |
| **모멘텀 가속/감속** | 레벨만 보여줌 (불리) | 히스토그램으로 직접 포착 (유리) |
| **전환 경고 (다이버전스)** | 직관적 (유리) | 히스토그램이 더 선행 (유리) |
| **과열 구간 익절 타이밍** | 70/80 레벨 명확 (유리) | 히스토그램 수축으로 간접 판단 (보통) |

> **실전 팁**: 둘을 병용할 때, RSI는 "지금 어느 레벨인지(위치)", MACD는 "모멘텀이 강해지는지 약해지는지(가속도)"로 역할을 나누면 서로 보완이 됩니다.

## MA + MACD 타임프레임 구성 전략

MA와 MACD를 같은 타임프레임에 같은 파라미터 구간으로 쓰면 사실상 같은 정보를 이중으로 보게 됩니다. 이 섹션에서는 두 지표가 서로 다른 정보를 제공하도록 구조화하는 방법을 다룹니다.

### MACD가 MA의 파생이라는 구조적 문제

MACD(12, 26, 9) 기본값의 실체:

| 구성 | 계산 |
|------|------|
| MACD선 | EMA(12) - EMA(26) |
| 시그널선 | EMA(MACD, 9) |
| 히스토그램 | MACD선 - 시그널선 |

**같은 타임프레임에서 MA(20, 60)와 MACD(12, 26, 9)를 같이 쓰면:**

- MACD(12, 26)는 사실상 **단기~중기 MA의 이격을 측정**하는 지표
- 여기에 MA(20, 60)를 얹으면 **비슷한 구간의 이동평균을 이중으로 보는 것**
- 결과적으로 신호가 거의 동시에 발생 → **확인 효과 없이 중복**

> MA와 MACD가 서로 다른 정보를 제공하려면, **타임프레임 또는 파라미터 구간을 의도적으로 분리**해야 합니다.

---

### 분리 방식 A. 멀티 타임프레임

> **MA는 상위 TF에서 레짐 판단, MACD는 하위 TF에서 타이밍 포착**

```
상위 타임프레임 (레짐/방향)
    MA 배열로 장세 분류
    예: 주봉 MA(20, 60, 120) → "지금이 상승 레짐인가?"
              ↓
하위 타임프레임 (진입/청산 타이밍)
    MACD로 모멘텀 확인
    예: 일봉 MACD(12, 26, 9) → "지금 진입할 모멘텀인가?"
```

**대표 조합 예시:**

| 매매 스타일 | MA 타임프레임 | MACD 타임프레임 | 비율 |
|------------|-------------|----------------|------|
| 스윙 (수일~수주) | 주봉 MA(20, 60, 120) | 일봉 MACD(12, 26, 9) | 1:5~7 |
| 데이트레이딩 | 일봉 MA(20, 60) | 1~4시간봉 MACD(12, 26, 9) | 1:6~24 |
| 중장기 포지션 | 월봉 MA(12, 24) | 주봉 MACD(12, 26, 9) | 1:4 |

**실전 작동 방식:**
1. 주봉 MA 배열 확인 → "상승 레짐" 판정
2. 일봉으로 내려와서 MACD 골든크로스 + 0선 상향 대기
3. 조건 충족 시 진입 / SL은 주봉 MA 이탈 기준

**장점**: MA와 MACD가 완전히 다른 정보를 담게 됨 — 중복 없음

**단점**: 상위 TF 신호가 바뀔 때까지 하위 TF 신호를 무시해야 하는 규율 필요

---

### 분리 방식 B. 파라미터 구간 분리

> **같은 타임프레임 안에서 MA는 중장기, MACD는 단기 구간으로 분리**

**문제 의식**: 일봉에서 MA(20, 60, 120)를 쓰고 MACD(12, 26)를 같이 쓰면 12~26 구간이 MA 20과 겹침

**해결책**: MACD 파라미터를 MA 구간과 겹치지 않게 조정

| 조합 | MA 구간 | MACD 구간 | 특징 |
|------|---------|-----------|------|
| 예시 1 | MA(60, 120, 200) | MACD(5, 13, 5) | MA는 중장기, MACD는 단기(5~13일) 포착 |
| 예시 2 | MA(20, 60, 120) | MACD(3, 8, 5) | MA는 중기 이상, MACD는 초단기 담당 |

**파라미터 설계 원칙:**

> MACD의 장기값(26) < MA의 단기값(60) — 이 조건을 만족하면 역할이 분리됩니다.

**장점**: 하나의 차트에서 즉시 확인 가능, 직관적

**단점**: MACD 파라미터를 커스텀하면 기본값 신호와 달라서 백테스트/레퍼런스 참고가 어려움

---

### 분리 방식 C. 역할 분담 + 동일 TF (가장 실용적)

> **MA는 구조(추세/레짐), MACD는 히스토그램의 수축/확대 패턴만 읽는 도구로 한정**

많은 실전 트레이더들이 쓰는 방식입니다. MACD를 "MA의 중복"이 아닌 **히스토그램의 수축/확대 패턴만** 읽는 도구로 한정하면 중복 문제가 실질적으로 줄어듭니다.

| 신호 | MA가 담당 | MACD가 담당 |
|------|----------|------------|
| 추세 방향 | O | — |
| 진입 레짐 | O | — |
| 모멘텀 가속 | — | O (히스토그램 확대) |
| 모멘텀 감속 | — | O (히스토그램 수축) |
| 전환 경고 | MA 배열 변화 | 다이버전스 |

---

### 매매 스타일별 권장 구성

#### 스윙 트레이딩 (수일~수주 보유)

| 레이어 | 구성 |
|--------|------|
| **레짐 판단** (주봉) | MA: 20, 60, 120 주봉. 장기MA 우상향 + 배열 확인 |
| **타이밍 진입** (일봉) | MACD: 12, 26, 9 일봉. 0선 위 골든크로스 + 히스토그램 확대 확인 |
| **SL 기준** | 주봉 MA(60) 종가 이탈 |
| **TP 기준** | 일봉 히스토그램 수축 전환 시 분할 청산 |

#### 데이트레이딩

| 레이어 | 구성 |
|--------|------|
| **레짐 판단** (일봉 또는 4시간봉) | MA: 20, 60. 배열과 기울기로 방향 확정 |
| **타이밍 진입** (1시간봉 또는 15분봉) | MACD: 12, 26, 9. 0선 위에서 골든크로스 + 히스토그램 확대 |
| **SL 기준** | 하위 TF 스윙저점 또는 상위 TF MA 이탈 |
| **TP 기준** | 히스토그램 고점 대비 50% 수축 시 절반 청산 |

#### 중장기 포지션 트레이딩

| 레이어 | 구성 |
|--------|------|
| **레짐 판단** (월봉) | MA: 12, 24 월봉 (= 1년, 2년 추세). 장기 추세 방향만 확인 |
| **타이밍 진입** (주봉) | MACD: 12, 26, 9 주봉. 0선 돌파 + 히스토그램 확대 |
| **SL 기준** | 월봉 MA(12) 종가 이탈 |
| **TP 기준** | 주봉 히스토그램 다이버전스 출현 시 일부 청산 |

---

### 타임프레임 구성 시 흔한 실수

| 실수 | 해결책 |
|------|--------|
| 상위 TF 레짐이 하락인데 하위 TF MACD 골든크로스만 보고 진입 | 상위 TF MA 배열 확인을 **게이팅 조건**으로 고정. "상위 TF가 상승 레짐일 때만 하위 TF 신호 채택" |
| MACD 파라미터를 MA와 같은 구간으로 설정해 신호 중복 | 방식 A(TF 분리) 또는 방식 B(파라미터 분리) 중 하나 선택 |
| 너무 많은 TF를 동시에 보다가 신호 충돌로 판단 마비 | TF를 **딱 2개만** 사용 (상위: 레짐, 하위: 타이밍). 3개 이상은 충돌 해결 규칙을 먼저 설계 |
| 하위 TF 신호가 나왔는데 상위 TF 신호를 기다리다 기회 놓침 | 역할을 명확히 — 상위 TF는 "진입 허가/금지"만 결정, 세부 타이밍은 하위 TF에서 완전히 위임 |

---

### 최종 권장 구조 요약

```
┌─────────────────────────────────────────────────┐
│  레짐 레이어 (상위 TF)                           │
│  MA 배열 → 방향 확정 / 진입 허가 여부 결정       │
└─────────────────────┬───────────────────────────┘
                      │ 상승 레짐 확인 시에만 ↓
┌─────────────────────▼───────────────────────────┐
│  타이밍 레이어 (하위 TF)                         │
│  MACD 히스토그램 + 0선 → 진입/청산 시점 결정     │
└─────────────────────┬───────────────────────────┘
                      │
┌─────────────────────▼───────────────────────────┐
│  SL/TP 레이어                                    │
│  SL → 상위 TF MA 기준 (구조적 손절)              │
│  TP → 하위 TF 히스토그램 수축 기준 (타이밍 익절)  │
└─────────────────────────────────────────────────┘
```

> **핵심 요약**: MA와 MACD를 같은 타임프레임에 같은 파라미터 구간으로 쓰면 사실상 같은 정보를 두 번 보는 것입니다. **MA는 레짐(방향)을, MACD는 모멘텀(가속도)을 담당하게 역할을 나누고, 이를 타임프레임 분리 또는 파라미터 구간 분리로 구조화하는 것**이 핵심입니다.
