3. 이렇게 실행의 대상이 되는데 깊숙하고 심오한 곳에 있는 대상을 실행할 때요 모듈로 실행, 즉 python -m과 같이 __main__.py를 활용합니다
4. 폴더를 import할 때는요 __init__.py만 실행하고 __main__.py는 실행을 안합니다
5. 이제 앞으로 직접 cli 앱을 만들 때 __main__.py를 활용하면은요 관리도 편하고 이 파일의 정체를 모르는 사람들한테 건방도 한 번 떨어줄 수 있을 겁니다

기타 메시지:
python -m 명령어는 단일 파일에도 적용 가능함. 하지만 폴더와 __main__.py 구조를 사용하면, 라이브러리로 import될 때와 명령어로 직접 실행될 때의 동작을 명확히 분리할 수 있어 코드 관리에 더 유리함.
