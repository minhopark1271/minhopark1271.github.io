### 전환별 모멘텀 버스트 분석

모든 전환이 아니라 **특정 전환에서만** 유의미한 방향성 움직임이 관찰됩니다.

| 🌒→🌓 | 가격이 EMA55 위로 돌파 | 🌓×🌓: LONG RR 7.7, 하방 -0.3% | **높음 (상방)** |
| 🌕→🌗 | 가격이 EMA21 아래 이탈 | 🌕×🌗: 등락 -0.1%, 양쪽 RR 낮음 | 약함 (노이즈) |
| 🌑→🌒 | 가격이 EMA21 위로 교차 | 🌑×🌒: 등락 +0.1%, 데드캣 확률 | 약함 (노이즈) |

### 1D가 4H를 압도하는 증거

타임프레임 간 충돌 시 상위(1D) 방향이 승리합니다.

| 충돌 조합 | 1D 방향 | 4H 방향 | 실제 결과 | 승자 |
|-----------|---------|---------|----------|------|
| 🌕×🌑 (39구간) | 상승 | 하락 | **+0.8%** (상승) | 1D |
| 🌑×🌕 (49구간) | 하락 | 상승 | **-0.8%** (하락) | 1D |

이것은 "페이크 무브"가 아니라 **상위 타임프레임으로의 평균 회귀**입니다. 스캘핑에서는 1D 방향을 따르는 것이 4H 전환 방향을 따르는 것보다 안전합니다.

### 데드캣 바운스 필터

1D가 하락(🌑, 🌘)인 상태에서 4H 상승 전환은 대부분 데드캣입니다.

| 조합 | 평균등락 | 해석 |
|------|---------|------|
| 🌑×🌒 | +0.1% | 거의 무의미한 반등 |
| 🌑×🌓 | +0.2% | 약한 반등 |
| 🌑×🌕 | **-0.8%** | 4H 상승추세에도 결국 하락 |

**즉시 적용 가능한 규칙**: `1D ∈ {🌑,🌘}` AND `4H ∈ {🌒,🌓,🌕}` → **LONG 진입 금지**.

### 이벤트 트리거로서의 한계

- 고확신 전환(🌓×🌓, 🌘×🌘)은 4년간 각 13회만 발생 → **연간 ~3~4회**

- 15분 보유 제한으로 전체 -41.3% 노출은 불가능하나, 15분 내 -3~5% 급락은 현실적
- 마틴게일 4단계(3.29%) × 급락 -3% = **계정 -9.9%** 가능
- 대응: 🌑×🌑에서 마틴게일 2단계 제한, 일일 한도가 최후 방어선

**🌕×🌕 최대상방 +33.5% — 숏 포지션 위험**
- 15분 내 +3~5% 스파이크는 현실적
- 마틴게일 4단계(3.29%) × 3x 레버리지 × +5% = **계정 -4.9%**
- 🌕×🌕에서 숏은 허용하지 않으므로(LONG ONLY) 구조적으로 회피됨

### 전체 거래 가능 비중

| 등급 | 비중 | 거래 가능 여부 |
|------|------|:------------:|
| 🟢 허용 | 61% | 정상 거래 |
| 🟡 주의 (축소 포함 시) | 34% | 단일 진입 가능 |
| 🔴 금지 | 5% | 거래 중단 |

**50% 기준 충족**: 허용 레짐만으로 61%, 축소 포함 시 95% 시간 활용 가능.

## 분석 5 — Gate 시스템 통합 설계

### Gate 0 분리 vs Gate 2 통합

| 기준 | Gate 0 분리 ✓ | Gate 2 통합 ✗ |
|------|-------------|-------------|
| 관심사 분리 | 명확 (거시 4H vs 미시 1분) | 혼재 |
| 업데이트 주기 | 4H마다 1회 (효율적) | 매 틱마다 불필요한 재계산 |
| 영향 범위 | 전체 파이프라인에 오버라이드 전달 | Gate 2만 영향 |
| 테스트/디버그 | 레짐 단독 테스트 용이 | 혼재 로직 디버그 복잡 |
| 거래 전면 차단 | 방어 모드로 가능 | 방향 제한만 가능 |

**결론: Gate 0 독립 분리**

### 통합 아키텍처

```
[Gate 0] 레짐 프리필터 (4H마다 업데이트)
    │
    ├─ 공격 LONG  → "LONG만 허용" + TP/Size 배수 전달
    ├─ 공격 SHORT → "SHORT만 허용" + TP/Size 배수 전달
    ├─ 기본       → "양방향 허용" + 기본 파라미터
    └─ 방어       → 전체 차단
    │
    ▼
[Gate 1] 환경 필터 — ADX(10) > 25, 시간대(UTC 13~17, 00~02), ATR < 2σ
    │
    ▼
[Gate 2] 방향 필터 — 15m EMA(21) + 5m MACD + VWAP ← 레짐 허용 방향으로 제한
    │
    ▼
[Gate 3] 트리거 — 1m RSI(7) 과매도/과매수 탈출 + MACD 전환 + EMA 크로스 + Volume > MA(20)×1.5
```

### 타임스케일 정합성 원칙

**"레짐은 필터, Gate는 트리거"**

| 레이어 | 측정 대상 | 전환 빈도 | 역할 |
|--------|----------|----------|------|
| 4H EMA 레짐 | 수일 단위 추세 방향 | 하루 0~1회 | "어느 편이 바람을 등지고 있나" |
| 15분 EMA(21) | 시간 단위 미시 방향 | 하루 3~8회 | "지금 이 파도의 방향" |

레짐이 허용하지 않는 방향의 거래는 실행하지 않습니다. 레짐이 허용해도 Gate 1~3 통과 필수.

- 레짐 Long + Gate 2 Long → **고확신 진입** (포지션 +20%)
- 레짐 Long + Gate 2 중립 → **표준 진입**
- 레짐 Long + Gate 2 Short → **거래 안 함** (차단)

15분 EMA와 4H EMA 레짐은 타임프레임이 다르므로 **중복이 아닌 상호보완**. 둘 다 유지.

### 전환 시 쿨다운

4H 봉 완성 후 레짐 변경 시 **15분 쿨다운**. 미시 지표가 이전 레짐의 관성을 아직 반영하기 때문.

### 기대 성능

| 지표 | 기존 (레짐 무관) | Gate 0 적용 | 변화 |
|------|-----------------|------------|------|
| 거래 횟수/일 | 12 | ~7 | -42% |
| 가중 승률 | 78% | ~81% | +3%p |
| 건당 EV | +0.333% | +0.445% | +34% |
| 일 기대수익 | ~4.0% | ~3.1% | -23% |
| **혼조장 일수익** | ~1.5% | ~2.5% | **+67%** |
| Sharpe 비율 | 기준 | **+30~40%** | 리스크 조정 우위 |

> 일 절대 수익은 감소하지만, 건당 EV +34%, 리스크 노출 -42%, 혼조장 성과 +67%로 **리스크 조정 수익률이 개선**됩니다.

### 구현 우선순위

| Phase | 조건 | 적용 | 기대 효과 |
|-------|------|------|----------|
| **1 (즉시)** | — | 방향 필터 + 방어 모드만, 배수 전부 1.0 | 혼조장 손실 회피 |
| **2 (100건)** | 실거래 데이터 | 공격 모드 TP ×1.2~1.3 | 건당 수익 확대 |
| **3 (300건)** | 레짐별 승률 검증 | 사이즈 ×1.1~1.3 + 마틴게일 차등 | 자본 효율 최적화 |

> **과최적화 방지**: RR 7.7에 TP ×1.5를 부여하는 것은 과거 데이터 특수성일 수 있음. Phase 1에서는 **방향 제한만 적용**하고, 배수 조정은 실거래 검증 후 단계적 도입.

## 5개 에이전트 합의 사항

| 합의 항목 | 내용 |
|-----------|------|
| **1D가 4H를 압도** | 충돌 시 1D 방향이 승리 (🌕×🌑=+0.8%, 🌑×🌕=-0.8%) |
| **방어 > 공격** | 불리 구간 차단(-27~33%p 방지)이 유리 구간 부스트(+4~14%p)보다 효과 큼 |
| **허용만으로 충분** | 정렬 조합 6~7개가 전체 61% → 거래 기회 충분 |
| **마틴게일은 방향 명확 시만** | 혼재/방어 구간 마틴게일 = 손실 증폭기 |
| **과최적화가 최대 리스크** | 소표본 레짐에 세밀한 배수 부여는 위험 |

## 즉시 적용 가능한 3가지 규칙

### 규칙 1: 방향 필터

```
공격 LONG 레짐 → Short 시그널 무시
공격 SHORT 레짐 → Long 시그널 무시
방어 레짐 → 모든 시그널 무시
```

### 규칙 2: 데드캣 바운스 필터

```
1D ∈ {🌑, 🌘} AND 4H ∈ {🌒, 🌓, 🌕} → LONG 진입 금지
```

### 규칙 3: 🌑×🌑 마틴게일 제한

```
레짐 = 🌑×🌑 → 마틴게일 최대 2단계 (급반등 방어)
```

## 핵심 결론

레짐 시스템의 가장 큰 가치는 "더 잘 치는 것"이 아니라 **"안 칠 때를 아는 것"**입니다.

- 방어 모드(20%)에서의 거래 중단이 전체 시스템 안정성을 구조적으로 개선
- 공격 모드에서의 파라미터 확대는 검증 후 단계적으로 도입
- 실전 1순위는 **방향 필터 + 데드캣 필터 + 마틴게일 차등** 3가지
