## 개요

Vercel은 프론트엔드 애플리케이션(특히 React, Next.js, 정적 웹사이트) 배포를 위한 클라우드 플랫폼이다. Next.js를 만든 팀에서 개발했으며, Vue.js, Angular, Svelte 등 다양한 프레임워크와 정적 사이트, 서버리스 함수 배포도 지원한다.

### 핵심 특징

- **서버리스 아키텍처**: 트래픽에 따라 자동 스케일링 (유휴 시 zero 스케일링)
- **글로벌 엣지 네트워크**: 전 세계 PoP(Point of Presence)를 통한 저지연 서비스
- **Git 통합**: 커밋/PR마다 자동 배포 및 프리뷰 URL 생성
- **Zero Config**: 대부분의 프레임워크에서 설정 없이 바로 배포

## 배포 방식

### 1. Git 연동 배포 (가장 일반적)

```bash
# GitHub/GitLab/Bitbucket 저장소를 Vercel에 import
# → 커밋하면 자동으로 배포됨
git push origin main
```

- 메인 브랜치 푸시 → Production 배포
- PR/브랜치 푸시 → Preview 배포 (고유 URL 발급)

### 2. Vercel CLI 배포

```bash
# Vercel CLI 설치
npm i -g vercel

# 프로젝트 디렉토리에서 배포
vercel

# Production 배포
vercel --prod
```

- Git 연결 없이도 배포 가능
- 로컬에서 빠르게 테스트 배포할 때 유용

### 3. Deploy Hooks

```bash
# 고유 URL로 POST 요청하면 배포 트리거
curl -X POST https://api.vercel.com/v1/integrations/deploy/xxx
```

- CI/CD 파이프라인 연동 시 유용
- Git 저장소 연결 필수 (단, 새 커밋 불필요)

## 인프라 아키텍처

Vercel에 서비스를 배포하면 아래와 같은 인프라가 자동으로 구성된다.

### 프론트엔드 애플리케이션이란?

사용자가 브라우저에서 보고 상호작용하는 모든 것을 말한다.

| 유형 | 설명 | 예시 |
|------|------|------|
| 정적 웹사이트 | HTML/CSS/JS 파일 그대로 제공 | 블로그, 포트폴리오, 랜딩페이지 |
| SPA | JS로 동적 렌더링하는 앱 | React, Vue, Angular 앱 |
| SSR 웹앱 | 서버에서 HTML 생성 후 전달 | Next.js, Nuxt.js 앱 |

### 렌더링 방식별 인프라 활용

| 렌더링 방식 | 설명 | 인프라 활용 |
|-------------|------|-------------|
| **SSG** (Static Site Generation) | 빌드 시 HTML 생성 | Edge CDN에서 즉시 응답 (가장 빠름) |
| **ISR** (Incremental Static Regeneration) | 정적 + 주기적 재생성 | Edge CDN + 백그라운드 재생성 |
| **SSR** (Server-Side Rendering) | 요청 시 HTML 생성 | Serverless Function 실행 |
| **CSR** (Client-Side Rendering) | 브라우저에서 렌더링 | Static JS 전송 후 클라이언트 처리 |

### 개발자가 관리하지 않아도 되는 것들

Vercel을 사용하면 다음 인프라 요소들이 자동으로 관리된다:

| 자동 관리 항목 | 설명 |
|----------------|------|
| 서버 프로비저닝 | 서버 생성/삭제/스케일링 자동화 |
| SSL 인증서 | Let's Encrypt 자동 발급/갱신 |
| CDN 구성 | 전 세계 엣지 캐싱 자동 설정 |
| 로드 밸런싱 | 트래픽 분산 자동 처리 |
| DDoS 방어 | 악성 트래픽 자동 차단 |
| 배포 롤백 | 원클릭 이전 버전 복원 |

## 주요 기능

### Serverless Functions

백엔드 API를 서버리스 함수로 처리:

```javascript
// api/hello.js
export default function handler(req, res) {
  res.status(200).json({ message: 'Hello World' });
}
```

- `/api/*` 경로에 파일 배치하면 자동으로 API 엔드포인트 생성
- Node.js, Go, Python, Ruby 지원

### Edge Functions

네트워크 엣지에서 실행되는 저지연 함수:

```javascript
// middleware.js
export function middleware(request) {
  // 사용자 위치 기반 리다이렉트 등
  return NextResponse.next();
}
```

- 미들웨어, A/B 테스트, 인증 등에 활용
- 전 세계 엣지에서 실행되어 매우 빠름

### 보안 기능

| 기능 | 설명 |
|------|------|
| Automatic HTTPS | 무료 SSL 인증서 자동 발급 |
| DDoS Protection | 매주 수십억 건의 악성 요청 차단 |
| WAF | SQL Injection, XSS 등 공격 방어 |
| IP Blocking | 특정 IP 차단 및 Rate Limiting |

규정 준수: SOC 2 Type 2, ISO 27001, PCI DSS, GDPR, HIPAA

## 2025 신기능 (Vercel Ship 2025)

### Rolling Releases

- 새 배포를 사용자 일부에게만 점진적 롤아웃
- 모니터링 및 롤백 컨트롤 내장
- 커스텀 라우팅 불필요

### Microfrontends

- 대형 앱을 독립 배포 가능한 작은 단위로 분할
- 팀별 독립 빌드/테스트/배포 가능
- Vercel이 통합 및 라우팅 처리

### Sandbox

- 최대 45분 실행 가능한 격리된 microVM 환경
- Vercel 외 플랫폼에서도 사용 가능한 독립 SDK
- Active CPU 기반 과금으로 비용 절감

### BotID

- 보이지 않는 CAPTCHA로 봇 차단
- 사용자 경험 저하 없이 보안 강화

## Vercel vs Streamlit 비교

| 구분 | Vercel | Streamlit |
|------|--------|-----------|
| **주요 용도** | 프론트엔드/React/Next.js 앱 | Python/ML/데이터 사이언스 앱 |
| **대상 사용자** | 웹 개발자 | 데이터 과학자, ML 엔지니어 |
| **필요 기술** | JavaScript/React | Python만 |
| **확장성** | 높음 (서버리스) | 제한적 |
| **UI 커스터마이징** | 완전한 제어 | 제한적 (빠른 프로토타입용) |
| **ML 연동** | API를 통해 연동 | 네이티브 (TensorFlow, PyTorch 등) |
| **배포 복잡도** | Git 연동만으로 간단 | Streamlit Cloud로 간단 |
| **트래픽 처리** | 대규모 트래픽 최적화 | 내부 대시보드/소규모 앱에 적합 |

### 언제 무엇을 선택할까?

**Vercel 선택**:
- React/Next.js 기반 웹 애플리케이션
- 대규모 트래픽이 예상되는 프로덕션 서비스
- 커스텀 UI/UX가 중요한 경우
- Firebase, Supabase, PostgreSQL 등 다양한 백엔드 연동 필요

**Streamlit 선택**:
- Python 기반 데이터 분석/ML 모델 데모
- 빠른 프로토타이핑 및 내부 대시보드
- JavaScript 학습 없이 웹앱 만들기
- Pandas, TensorFlow, Hugging Face 등 직접 연동

### 주의사항

- Vercel은 Flask나 Streamlit을 직접 지원하지 않음
- Streamlit 앱을 Vercel에 배포하려면 상당한 변경 필요
- Streamlit 앱은 Streamlit Cloud, Heroku, DigitalOcean이 더 적합
