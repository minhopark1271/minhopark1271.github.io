## 시장 환경: 무엇이 바뀌고 있는가

### 창업 진입장벽의 붕괴

클라우드, AI, 오픈소스가 동시에 성숙하면서 창업의 3대 장벽이 해체되고 있습니다.

| 과거의 장벽 | 현재 상태 |
|------------|----------|
| **자본** | 종량제 API + 클라우드로 초기 비용 급감 |
| **유통** | PLG·앱스토어로 영업 조직 없이 수백만 도달 |
| **코드** | 오픈소스 + 생성형 AI로 개발 장벽 극적 하락 |

Gamma 사례가 단적입니다. 6만 사용자 스타트업이 AI-native로 재구축한 뒤 3년 만에 7,000만 사용자, 21억 달러 기업가치를 달성했습니다. **"직원 수와 임팩트 간의 관계가 영구적으로 깨졌다"**는 선언이 나올 정도입니다.

### 기능 구현 비용의 무의미화

MVP 수준의 기능 구현 비용이 <span>$</span>200K에서 <span>$</span>20~100으로 떨어졌습니다. 누구나 AI로 경쟁 제품을 빠르게 만들 수 있는 상황에서 **"기능 기반 차별화는 무의미"**해졌습니다. 새로운 기준은 MVP가 아닌 **MLP(Minimum Lovable Product)** — 사용자가 감정적으로 사랑할 수 있는 최소 제품입니다.

## 핵심 트렌드 ①: 코파일럿에서 오토파일럿으로

Sequoia Capital이 제시한 가장 강력한 프레임워크입니다.

### 지능(Intelligence) vs 판단(Judgment)

- **지능**: 규칙이 복잡하지만 규칙 자체인 작업 (코딩, 양식 작성, 데이터 처리)
- **판단**: 경험·감각에 기반한 의사결정 (전략, 협상, 조직 설계)

지능 비율이 높은 업무일수록 AI 자동화가 빠르게 진행됩니다.

### 전환의 방향

| 단계 | 모델 | 판매 대상 | 예시 |
|------|------|----------|------|
| **코파일럿** | 전문가에게 도구 판매 | 개발자, 회계사 | GitHub Copilot, Cursor |
| **오토파일럿** | 고객에게 결과물 판매 | CFO, CEO | AI가 보험 직접 중개, 코드 직접 납품 |

**SI/컨설팅 관점의 시사점**: 기존 SI는 "개발 인력을 파견"하는 코파일럿 모델이었습니다. Agentic SI는 **"결과물(완성된 시스템)을 직접 납품"**하는 오토파일럿 모델로 전환할 기회입니다.

## 핵심 트렌드 ②: 서비스의 소프트웨어화 — 양측 관점

### 낙관론 (Sequoia)

AI로 외주화된 지능 작업에 진입 → 기존 벤더 교체 → 데이터 축적 → 내부 업무 확장. 보험 중개(<span>$</span>140-200B), 채용(<span>$</span>200B+), 공급망(<span>$</span>200B+), **경영 컨설팅(<span>$</span>300-400B)** 등 수십~수백조 원 규모의 시장이 열립니다.

### 현실론 (Better Tomorrow Ventures)

- 전문 서비스 지출의 상당 부분은 **"책임 전가, 신뢰성, 자격 인증"**에 대한 대가 → AI로 대체 불가
- AI로 얻은 마진 우위는 일시적 → 경쟁사 채택과 고객의 비용 전가 요구로 빠르게 잠식
- 소프트웨어 수준 마진(70-85%)은 어렵지만, **50% 이상 마진 + 책임 소유권**이면 충분한 가치 창출 가능

### 종합 판단

서비스는 완전한 소프트웨어가 되지는 않지만, **"더 소프트웨어에 가까워질 것"**이며, 이 시장 규모에서는 그것으로 충분합니다. Agentic SI/컨설팅은 **판단(Judgment) 영역에서의 신뢰와 책임을 유지하면서, 지능(Intelligence) 영역을 에이전트로 자동화**하는 하이브리드 모델이 적합합니다.

## 핵심 트렌드 ③: 에이전트 코딩의 실전과 교훈

### Lablup 사례 — 130억 토큰, 100만 줄

Backend.AI:GO를 40일간 개발하며 약 100만 줄 코드를 생성한 실전 경험에서 도출된 원칙:

1. **토큰 효율성이 경쟁력** — 에이전트 코딩 시대에 토큰 사용량이 IT 기업의 핵심 역량
2. **Harness가 핵심** — 모델 자체보다 모델을 제어하는 로직(harness)이 성능을 결정
3. **결과물 직진 금지** — 컨텍스트 구축 우선, 결과물 생성 장치 구축 후 반복(iteration)
4. **자동화의 본질** — 결과물이 아닌 "생성하는 장치"를 만드는 것

### Amazon의 교훈 — 시니어 승인 의무화

AI 코딩 도구로 인한 연속 장애(쇼핑앱 6시간 다운, AWS 비용 계산기 13시간 장애) 후, 신입/중급 엔지니어의 AI 코드 변경에 **시니어 엔지니어 사전 승인을 의무화**했습니다.

**시사점**: AI가 코드를 생성하는 속도는 빨라졌지만, **검증과 품질 보증은 여전히 인간의 판단 영역**입니다. 이것이 Agentic SI/컨설팅의 핵심 가치 제안이 됩니다.

### 자율 에이전트 검증 파이프라인

"잠자는 동안 실행되는 에이전트" 구축 사례에서, 가장 중요한 문제는 **"같은 AI가 코드 작성과 테스트를 모두 하면 오류를 놓침"**이라는 자기 축하 기계(Self-congratulation machine) 현상입니다.

해결책: 코드 작성 전 **수용 기준(Acceptance Criteria)을 먼저 정의**하고, 별도 에이전트가 검증하는 분리 파이프라인 구축. **"스펙 정의가 프롬프트 작성보다 어렵지만, 신뢰성 확보의 필수 절차"**입니다.

## 사업 아이템별 적용

### 1. Agentic SI (소프트웨어 개발 대행)

**기존 SI와의 차이점**:

| 항목 | 기존 SI | Agentic SI |
|------|---------|------------|
| 핵심 자산 | 개발 인력 수 | 에이전트 파이프라인 + 검증 체계 |
| 납품물 | 코드 + 문서 | 동작하는 시스템 + 자동화된 검증 |
| 마진 구조 | 인건비 기반 (15-25%) | 토큰 비용 기반 (50%+ 가능) |
| 확장 방식 | 인력 충원 | 에이전트 병렬화 |

**핵심 역량**: Harness 설계, 컨텍스트 엔지니어링, 수용 기준 정의, 시니어 레벨 검증

### 2. 엔터프라이즈 컨설팅

Sequoia가 말한 **경영 컨설팅 시장(<span>$</span>300-400B)**은 판단 중심이라 완전 자동화가 어렵지만, 지능 영역의 자동화로 **"더 적은 인원으로 더 깊은 분석"**이 가능합니다.

**컨설팅 서비스 재설계**:
- **지능 영역 (에이전트화)**: 데이터 수집·분석, 벤치마킹, 보고서 초안, 코드 리뷰
- **판단 영역 (인간 유지)**: 전략 의사결정, 이해관계자 조율, 조직 변화 관리, 책임 소유

BTV의 지적대로 **"책임 전가, 신뢰성, 자격 인증"**은 AI로 대체 불가능합니다. 이것이 컨설팅의 지속 가능한 가치입니다.

### 3. 스타트업 팀빌딩·액셀러레이팅

**"기술 공동창업자가 정말 필요한가?"** — 이 질문이 현실이 된 시대에, 팀빌딩의 본질이 바뀝니다.

**새로운 팀빌딩 프레임워크**:

| 역할 | 과거 | 현재 |
|------|------|------|
| 코딩 | CTO/개발팀 필수 | 에이전트 + 시니어 1명으로 가능 |
| 제품 | PM이 스펙 작성 | 수용 기준 정의 능력이 핵심 역량 |
| 디자인 | 디자이너 필수 | MLP 관점의 감성 설계 역량 중요 |
| 검증 | QA팀 | 자동화 검증 파이프라인 |

**액셀러레이팅 서비스**:
- 에이전트 파이프라인 세팅 (Harness + 검증 체계)
- MLP 기준의 제품 코칭
- "생성 장치를 만드는 법" 교육

### 4. 협업문화 컨설팅

에이전트와 인간이 협업하는 새로운 문화가 필요합니다.

**핵심 이슈들**:
- **검증 프로세스**: Amazon 사례처럼, 시니어 승인은 필수지만 병목이 되면 안 됨
- **스펙 퍼스트 문화**: 프롬프트보다 수용 기준 정의가 중요 → 조직 전체의 역량
- **토큰 거버넌스**: 토큰 사용량이 곧 비용이자 경쟁력 → 측정·최적화 문화
- **반취약한 정체성**: "직업"이 아닌 "학습·적응 능력"으로 정체성 재정의

## 방어 전략: 새로운 해자(Moat)

모든 창업자가 같은 AI API에 접근할 수 있을 때, 기술 우위는 소멸합니다. 방어력은 모델 외부에서 확보해야 합니다.

### 6가지 해자

| 해자 | Agentic SI/컨설팅 적용 |
|------|----------------------|
| **독점 데이터** | 프로젝트 수행 중 축적되는 산업별 컨텍스트, 검증 기준 라이브러리 |
| **네트워크 효과** | 에이전트가 네트워크 노드가 되어 인간 불가능한 규모로 상호작용 |
| **워크플로 고착화** | 고객사에 에이전트 파이프라인이 깊이 통합되면 전환 비용 발생 |
| **브랜드 신뢰** | 검증·책임 소유의 트랙 레코드 |
| **유통 채널** | 기존 고객사를 통한 확장 (SI → 컨설팅 → 액셀러레이팅) |
| **도메인 전문성** | 특정 산업의 판단(Judgment) 영역 전문성 |

에이전트 시대에도 **네트워크 효과는 유효**합니다. 오히려 에이전트가 네트워크 노드로 참여하면서, UI 없는 온보딩으로 참여 장벽이 제거되고, 인간만으로는 불가능했던 규모의 네트워크 효과가 가능해집니다.

## 실행 원칙

자료들에서 반복적으로 등장하는 핵심 원칙을 정리합니다.

1. **가치 창출에만 집중** — 경쟁 게임을 피하고, 소비보다 많은 가치를 만드는 데 몰두 (geohot)
2. **학습 속도가 최적화 대상** — 완벽한 계획이 아닌, 실제 사용자 행동 신호를 가장 빨리 얻는 것이 우선 (창업의 새로운 규칙)
3. **결과물이 아닌 생성 장치를 만들어라** — 코드가 아닌 코드를 만드는 시스템을 구축 (Lablup)
4. **스펙 정의가 프롬프트보다 중요** — AI 주도 개발에서 신뢰성의 근원 (자율 에이전트 검증)
5. **판단 영역의 책임을 소유하라** — AI가 대체할 수 없는 마지막 가치 (Sequoia + BTV 종합)
6. **사랑받는 제품을 만들어라** — 기능이 아닌 감정적 연결이 차별화 (MLP)

---

## 결론

**"재능과 자원은 상품화되고 있으며, 남은 부분은 마음(heart)"** — MLP 시대의 선언은 Agentic SI/컨설팅에도 그대로 적용됩니다.

코드를 짜는 능력은 상품화되었습니다. 남은 것은 **무엇을 만들지 판단하는 능력, 만든 것을 검증하는 책임, 그리고 고객이 사랑할 수 있는 경험을 설계하는 감각**입니다. 이 세 가지가 Agentic SI/컨설팅/협업문화 사업의 핵심 가치 제안입니다.
