# Blog Writer

기술, 투자, 경영, 기타, 또는 개인 주제의 블로그 포스트를 마크다운 형식으로 작성합니다.

## 블로그 경로

| 카테고리 | 경로 | parent 값 |
|---------|------|-----------|
| 기술 블로그 | `docs/development/` | 개발 |
| 투자 블로그 | `docs/investment/` | 투자 |
| 경영 블로그 | `docs/startup/` | 경영 |
| 기타 블로그 | `docs/etc/` | 기타 |
| 개인 블로그 | `docs/personal/` | 개인 |

## 데이터 소스

사용자로부터 다음 중 하나의 소스를 받습니다:

1. **복사/붙여넣기**: 사용자가 직접 붙여넣은 텍스트 정보
2. **URL**: 제공된 웹페이지 URL → WebFetch로 내용 수집
3. **조사 요청**: 주제 제시 → WebSearch로 직접 조사 후 작성

## 작성 절차

### 1. 카테고리 확인
- 기술 관련 (프로그래밍, 도구, 라이브러리 등) → `docs/development/`
- 투자 관련 (금융, 트레이딩, 시장 분석 등) → `docs/investment/`
- 경영 관련 (사업, 조직, 재무, 법률, 스타트업 등) → `docs/startup/`
- 기타 (위 카테고리에 속하지 않는 주제) → `docs/etc/`
- 개인 (개인적 기록, 일기, 메모 등) → `docs/personal/` (**항상 `published: false`** — 절대 publish하지 않음)

### 2. nav_order 결정
해당 폴더의 기존 파일들을 확인하여 마지막 nav_order + 1 사용:
```bash
ls docs/{category}/
```

### 3. 파일명 결정
- 영문 소문자, 하이픈(-) 사용
- 주제를 간결하게 표현
- 예: `lstm-prediction.md`, `funding-rate.md`

### 4. 마크다운 작성

## 마크다운 템플릿

```markdown
---
title: {키워드 1-2단어}
parent: {개발 또는 투자 또는 경영 또는 기타 또는 개인}
nav_order: {숫자}
description: {150자 이내 SEO 설명 - 포스트 핵심 내용 요약}
---

# {간결한 헤드라인}
{:.no_toc}

{간단한 설명 1-2줄}

### Link

- [링크 제목](URL)

## 목차
{:.no_toc}

1. TOC
{:toc}

---

## 첫 번째 섹션

내용...

### 하위 섹션

세부 내용...

---

## 두 번째 섹션

내용...
```

## 작성 규칙

### 필수 요소
- frontmatter: `title`, `parent`, `nav_order`, `description`
- 제목 바로 아래 `{:.no_toc}` 추가
- 목차 섹션 포함
- **개인 카테고리(parent: 개인)는 반드시 `published: false` 추가** — 절대 publish하지 않음

### 민감 정보 자동 검출 (CRITICAL)
작성 완료 후, 포스트 내용에 아래 항목이 포함되어 있으면 frontmatter에 `unpublished: true`를 자동 추가:
- **개인 식별 정보**: 본명, 전화번호, 주소, 주민등록번호, 계좌번호, 카드번호
- **금융 민감 정보**: 구체적 자산 금액, 계좌 잔고, 세금 신고 내역, 대출 상세, 연봉/소득 금액
- **의료/법적 정보**: 진단명, 처방 내역, 소송 상세, 합의금 금액, 보험 상세
- **사적 관계 정보**: 가족/지인 실명과 함께 언급된 사적 내용, 갈등/분쟁 상세
- **보안 정보**: API 키, 비밀번호, 서버 주소, 토큰, 인증 정보

검출 시 자동으로 `unpublished: true` 추가하고, 사용자에게 해당 항목을 안내하여 확인 요청.
개인 카테고리가 아닌 경우에도 위 항목이 발견되면 적용.

### 제목 규칙 (CRITICAL)

**`title` (frontmatter)과 `#` (headline)은 다른 역할이며 반드시 분리 작성합니다.**

- **title** (frontmatter): 사이드바 네비게이션에 표시되는 **구분자**
  - 핵심 키워드 1~3단어 (15자 이내 필수)
  - 문장형 금지, 조사/수식어 금지
  - 예: `VCP`, `투자 성과 지표`, `다우존스`, `Funding Rate`
  - 반례: ~~`Volatility Contraction Pattern`~~, ~~`백테스트 성과 지표 완전 가이드`~~, ~~`다우존스 산업평균지수 (DJIA)`~~
- **# headline**: 본문 최상단 제목
  - title보다 구체적이되 간결하게 (20자 이내 권장)
  - 예: `Sharpe Ratio 해석 가이드`, `Funding Rate 구조와 활용`

### SEO 가이드라인
- **description** (필수): 150자 이내로 포스트 핵심 내용 요약
  - 검색 결과 및 소셜 미디어 공유 시 표시됨
  - 핵심 키워드를 자연스럽게 포함
  - 완전한 문장으로 작성
  - 예: "Python uv 패키지 매니저의 설치 방법과 pip 대비 장점, 주요 명령어 사용법을 설명합니다."

### 스타일 가이드
- **언어**: 한국어 기본 (기술 용어는 영문 병기 가능)
- **섹션 구분**: `---` 수평선으로 주요 섹션 분리
- **코드 블록**: 언어 명시 (```python, ```bash 등)
- **표**: 비교나 정리에 마크다운 테이블 활용
- **볼드/리스트**: 핵심 개념 강조, 항목 나열에 활용
- **달러 기호 (CRITICAL)**:
  - `$`는 MathJax 인라인 수식(`$...$`)으로 파싱됨. 같은 블록에 `$`가 2개 이상 있으면 수식으로 깨짐
  - 통화 표기 시 반드시 `<span>$</span>`으로 감싸기 (예: `<span>$</span>100`)
  - `&#36;`이나 `\$`는 효과 없음
  - **표(테이블) 안에서도 동일 적용**: `| <span>$</span>5,000 | <span>$</span>10,000 |`
  - **작성 완료 후 검수**: 파일 전체에서 `$` 문자를 검색하여, 수식 용도가 아닌 모든 `$`를 `<span>$</span>`으로 치환했는지 확인

### 내용 구조
1. 개념 정의/소개
2. 상세 설명 (예시 포함)
3. 실습/사용법 (해당시)
4. 요약/참고사항

## 예시 호출

```
"LSTM에 대해 블로그 글 작성해줘"
→ WebSearch로 조사 → docs/development/lstm.md 생성

"이 URL 내용으로 투자 블로그 작성해줘: https://example.com/article"
→ WebFetch로 수집 → docs/investment/{topic}.md 생성

"다음 내용으로 기술 블로그 작성해줘: [붙여넣은 내용]"
→ 내용 정리 → docs/development/{topic}.md 생성
```

## 작성 완료 후 (Commit & Push)

글 작성 완료 후 아래 절차를 **자동으로** 수행합니다. 사용자에게 매번 확인받지 않습니다.

### Step 1: 커밋 & 푸시

```bash
git add -A && git commit -m "{포스트 제목 요약}" && git push
```

### Step 2: 완료 안내

1. 생성된 파일 경로 안내
2. 커밋 해시 및 푸시 상태 안내
3. `unpublished: true`가 추가된 경우 해당 사유 안내
